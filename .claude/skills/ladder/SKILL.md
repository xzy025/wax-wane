# /ladder — 连板梯队 + 打板情绪 + 退潮研判

输出全市场（或指定主线）的**连板梯队、情绪阶段、退潮研判、次日盯盘清单**。复刻「首板涨停 + A股情绪周期」逻辑做盘面级研判（区别于个股 `/ta`、`/equity`）。

## 输入

`/ladder [主线关键词] [YYYYMMDD]`
- 例：`/ladder`（全市场）、`/ladder 铜箔`、`/ladder 算力`、`/ladder PCB 20260616`
- 关键词可选；不给则出全市场梯队。日期可选，默认最新交易日（接口返回 `data.qdate`，盘中为实时）。

## 步骤

1. **取数**（按 `.claude/skills/_shared/data-sources.md` 第 6 节，失败静默降级）：
   - 涨停池 `getTopicZTPool`（`pagesize` 拉大到 ~300）：`data.tc` 总数 + `pool[]` 每项 `n`名称/`c`代码/`lbc`连板数/`zbc`炸板次数/`hybk`板块/`zttj`(几天几板)/`zdp`涨幅。
   - 跌停池 `getTopicDTPool` → `tc`；炸板池 `getTopicZBPool` → `tc`（取不到则用涨停池里各股 `zbc` 估分歧）。
2. **梯队统计**：按 `lbc` 分布出梯队（如 4板×N、3板×N、2板×N、1板×N）；找**空间龙**（最高 `lbc`）。
3. **主线过滤**（给了关键词时）：`hybk` 行业分类较粗，需结合常识识别细分（如铜箔/覆铜板/电子布/树脂同属 PCB 产业链；CPO/光模块/算力设备同属算力链）——按板块 + 个股名 + 已知细分归类，输出该主线梯队（高度排序），标注题材人气龙是否在涨停池内（不在=分歧）。
4. **情绪阶段判定**（`.claude/skills/_shared/data-sources.md` 第 6 节速判 + `src/agent/tradingPatterns.ts` 的 `sentiment` 四阶段）：冰点/修复/高潮/退潮；并看晋级率、赚钱效应（涨停 vs 跌停、昨日涨停今表现）。
5. **退潮研判**：龙头分歧（人气龙/空间龙是否冲高回落、跌出涨停池、高 `zbc` 炸板）、高位炸板率、跌停是否抬头、梯队是否断层。
6. **输出**（见下），附免责声明。

## 输出格式

```
## 🪜 连板梯队 + 打板情绪 — [主线/全市场] — {YYYY/M/D}
> 全市场：涨停 {ZT} / 跌停 {DT} / 炸板 {ZB}；梯队 4板×N、3板×N、2板×N、1板×N

### 梯队（高度排序，标注 空间龙 / 细分 / 炸板次数）
| 高度 | 个股(代码) | 细分 | 分歧(炸板) |
...

### 情绪阶段：**冰点/修复/高潮/退潮**（依据：…）
- 晋级率 / 赚钱效应 / 龙头表现

### 退潮研判
- 龙头分歧、炸板率、跌停抬头、梯队断层 → 是否退潮 / 风险累积

### 次日盯盘清单（退潮确认信号）
- ① 人气龙是否破位 ② 空间龙是否炸板/断板 ③ 涨停回落+跌停抬头 ④ 赚钱效应转亏钱效应

> 数据自动生成，仅供研究参考，不构成投资建议。
```
